## 应用实例计费

AutoDL Art的应用实例支持按量计费和包年包月两种计费方式

### 按量计费

<Warning>注意：系统盘单独计费，不包含在应用实例计费中，请参考下方系统盘的计费规则</Warning>

计费规则：

1. 费用计算：应用实例开机开始计费，关机结束计费。费用= 时长x单价，时长=关机时间-开机时间，精确到秒，最低计费0.01元，单价以网页显示为准
2. 扣费时机：整点扣费一次和关机时扣费一次，扣费金额为扣费周期内的费用，使用时长精确到秒。如10:00开机，11:30关机，那么会在11:00扣费1小时费用和在11:30扣费0.5小时费用
3. 注意计费时长是以应用实例开关机时间为准，而不是以是否调用GPU计算时长为标准

### 包年包月

计费规则：

1. 包年包月为预付费，租用时一次性付清，具体价格以网页显示为准
2. 支持按日、按周、按月等租用GPU
3. 包年包月到期后，应用实例会关停（数据不受影响）
4. 支持续费，续费周期支持按日、按周、按月 。也可以转按量计费，请见下方转换计费方式

### 转换计费方式

按量计费和包年包月支持互转。如果包年包月转按量计费则会退款，退款规则为：计算包年包月期间已消费的费用，单价根据已使用时长来确定，比如包了1月仅使用8天后转按量计费那么会以包周（使用超过7天）的价格来计算已消费金额，那么这里已消费金额即=周单价 x (8天/7天)，如果这里是使用3天后即转按量计费那么已消费金额=日单价 x (3天/1天)，然后在扣除已消费金额时优先使用已支付的代金券，未能完全抵扣的再使用已支付的余额部分，最后未消费的余额部分退款。注意如果代金券抵扣后还有剩余不再退还代金券。从退款规则看，**最经济**的使用方式是：包年包月，然后如果不再使用后转按量退款处理，这样会按照更经济的单价计费。

## 系统盘计费

计费单价：请以网页显示的系统盘单价为准

计费规则：

1. 系统盘按日进行计费和结算。系统盘基础容量30GB，基础容量部分0.10元/日，系统盘支持扩容，扩容增加的费用以网页显示为准
2. 如果应用实例是按量计费，系统盘会在次日凌晨结算当天的费用，每日结算一次，**无论实例是否开机**，均会计费，如不再需要使用，在实例的更多操作中点击**缩容系统盘**或者直接**释放实例** 
3. 如果应用实例是包年包月，那么系统盘会计算在总费用中，租用时付清，实例到期后则会以按量计费实例一样的规则计费，即: 次日凌晨结算当天的费用，每日结算一次，**无论实例是否开机**
4. 如果账户余额不足时会优先保留应用实例数据产生计费，不会立即释放您的实例，如果账号欠费超50元，平台将保留删除数据的权利（考虑到数据的重要性，会谨慎考虑删除用户数据），如由于平台在欠费50元后未能及时删除数据导致持续扣费，超出欠费50元的部分我们将为您使用无门槛代金券补欠额(例如：欠费60元，那么可联系客服补10元代金券；如欠费20元那么将由用户自行承担)

<Note>如果扩容系统盘不再需要，可以在实例更多操作中点击缩容系统盘。然后当日的计费容量=当日最大的扩容容量，比如系统盘扩容为100GB，然后当天又缩容至10GB，那么当日以100GB计费。</Note>

## 镜像计费

可保存的镜像数量默认为20个，如需更多数量请联系客服。镜像计费以所有镜像存储总容量大小进行计费，与镜像数量无关。

计费单价：￥0.01/GB/日

计费规则：

系统将以当日(自然日)使用的 (最大容量-免费容量) 为计费容量。30GB以下免费，超出免费30GB容量的费用(元/日)= 超出容量(GB) × 0.01元/GB/日，扣费时间为次日凌晨扣除当日费用。如有超出免费容量，最低费用0.01元
如果账户余额不足时会优先保留数据产生计费，不会立即删除您的数据，但如果账号连续3个月未登录或欠费超50元，平台将保留删除数据的权利（考虑到数据的重要性，会谨慎考虑删除用户数据），如由于平台在欠费50元后未能及时删除数据导致持续扣费，超出欠费50元的部分我们将为您使用无门槛代金券补欠额(例如：欠费60元，那么可联系客服补10元代金券；如欠费20元那么将由用户自行承担)
举例：如果当日使用最大量为50GB，那么当日产生的费用 = (50 - 30) × 0.01 = 0.2元
